    HatchTech als PHP/Java Engineer (augustus 2019 – februari 2020):
    HatchTech is een bedrijf die zich specialiseert in het ontwikkelen van broedmachines voor kippen. Deze broedmachines worden beheerd door een lokale computer. Vanuit meerdere klanten is de wens gekomen om deze data ook in te kunnen zien op bijvoorbeeld hun mobiele telefoon. Doordat deze data lokaal op deze computer stond was dit nog niet mogelijk.

    Bij HatchTech heb ik mij verdiept in het ontwikkelen van zowel een backend applicatie om de data op te slaan, als een frontend applicatie om de data te tonen. Om deze informatie van de lokale computer naar het database te krijgen heb ik een applicatie geschreven in Java. Deze draait op de achtergrond en eens per 5 minuten haalt deze applicatie de nieuwe gegevens op uit het lokale database (dBase 7) en stuurt deze gegevens naar de PHP-applicatie. De PHP-applicatie zet de gegevens vervolgens in een MySQL database.

    De PHP-applicatie is ook voorzien van een simpele frontend applicatie. Deze is gemaakt met VueJS. Hiermee kunnen gebruikers gemakkelijk inloggen en hun gegevens inzien. Deze applicatie is responsive waardoor deze goed werkt op zowel een mobiele telefoon als op een computer of tablet.

    De uitdaging bij dit project is dat er op sommige locaties geen stabiel internet aanwezig is. Hierdoor moet er altijd rekening gehouden worden dat het versturen van de data fout kan gaan.

    Een andere uitdaging bij dit project was dat er gebruik werd gemaakt van een ouder database model waardoor er geen gemakkelijke manier was om de data uit te lezen. Hiervoor heb ik een file structure gevonden van de database (zie https://www.dbase.com/Knowledgebase/INT/db7_file_fmt.htm) en zelf een library geschreven om deze data uit te lezen.

    Aan het begin van mijn opdracht heb ik mij bezig gehouden met het beveiligen van het intranet van HatchTech. In mijn eerste weken viel het mij op dat deze niet heel goed beveiligd was. Het was mogelijk om bijvoorbeeld bestanden te downloaden (bijvoorbeeld index.php) door de url aan te passen of het database uit te lezen door middel van SQL injections.

    Sportunity in als PHP Engineer (juni 2018 – december 2018)
    Sportunity is een bedrijf dat sportgerelateerde apps maakt. Bij Sportunity heb ik mij ingezet om een API te ontwikkelen. Deze API werd gebruikt om de Android app, de iOS app en het web-dashboard met data te voorzien. Hier zijn meerdere API’s voor ontwikkeld, omdat hier gebruik werd gemaakt van een Single Sign On (SSO). Hierdoor kan een klant gemakkelijk gebruik maken van alle apps die door Sportunity worden gereleased.

    Het opzetten van de SSO was uitdagend om te doen, omdat hierover goed overlegd moest worden welke data opgeslagen gaat worden bij het account op de SSO en welke data er bij de API zelf wordt opgeslagen. Er is gekozen voor een SSO, omdat het ontwikkelen van de API’s hierdoor gemakkelijker werd. Dit kwam omdat het inloggen, registreren en de wachtwoord vergeten functionaliteit al ontwikkeld waren. Ook is hiervoor gekozen omdat dit voor de klant gemakkelijker is wanneer deze een andere sport wilt gaan doen.

    Omdat er verschillende API’s zijn met elk hun eigen benodigdheden, maar ook met overeenkomsten, is ervoor gekozen om microservices te maken. Een voorbeeld hiervan is het versturen van de notificaties. Deze worden niet per project afgehandeld, maar naar een microservice gestuurd. Hiervandaan worden dan alle notificaties naar de gebruikers verzonden.

    Gebruikte technieken:
    Java / Spring / Hibernate / Maven / Gradle / PHP / JavaScript / Sass / CSS / HTML / ECMAScript 6 / SCRUM / Agile / MySQL / Docker / Gitlab / dBase 7

    De Online Fabriek

    Functie: Medior PHP Ontwikkelaar
    Locatie: Planetenveld 35, Zeewolde

    Werkzaamheden:

    Bij de Online Fabriek worden webapplicaties, die zijn gemaakt door middel van het Laravel framework, ontwikkeld en onderhouden. Bij deze projecten is er vaak een goede samenwerking met de klant en/of externe partijen vereist.

    Een voorbeeld van een project waaraan ik heb gewerkt is ETA-monitor. Met deze applicatie is het mogelijk om pakketten te traceren voor bedrijven die deze mogelijkheid zelf niet hebben. Dit kan vanuit de verzender, maar ook vanuit de bezorgende partij. Zodra een klant de link ontving om het pakket te traceren, moest de look en feel aangepast worden voor deze klant. Zo moest het juiste logo erboven komen en/of de kleuren aangepast worden.

    Een ander project waar ik aan heb gewerkt is een enquête website. Wanneer mensen zich hadden aangemeld en er een enquête beschikbaar was, werd een link voor deze enquête naar hen verstuurd. Wanneer deze was ingevuld, kregen deze personen hier een bepaald bedrag voor. Bij dit project was het belangrijk dat er snel en gemakkelijk door de data gefilterd kon worden. Bij de applicatie die gebouwd was kon dit tot enkele minuten oplopen en het was de wens dat dit sneller ging. Hiervoor is Elasticsearch gebruikt. Door gebruik te maken van Elasticsearch hebben wij dit van enkele minuten weten terug te brengen naar enkele seconden.

    Bij de Online Fabriek heb ik mij ook bezig gehouden met het instellen van Bitbucket Pipelines. Hiermee is het mogelijk om de unittests van onze applicaties al te draaien zodra er een feature branch wordt gepushed of als er bijvoorbeeld een pull request wordt aangemaakt. Als de unittests dan falen weet de ontwikkelaar meteen dat er iets mis is en kan hier meteen actie op ondernemen.

    Laravel / PHP / PHPunit / JavaScript / HTML / CSS / Sass / GIT / ECMAScript 6 / MySQL / Elasticsearch / Redis / SCRUM / Agile / Continuous Integration
